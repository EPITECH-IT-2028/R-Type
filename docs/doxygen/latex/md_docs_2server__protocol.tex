\hypertarget{md_docs_2server__protocol_autotoc_md16}{}\doxysection{\texorpdfstring{1. Introduction}{1. Introduction}}\label{md_docs_2server__protocol_autotoc_md16}
The {\bfseries{R-\/\+Type Network Protocol (RTNP)}} defines how clients and the game server exchange structured {\bfseries{binary messages over UDP}}. ~\newline
 It supports synchronization of player state, enemies, projectiles, and combat events in a real-\/time environment.

The protocol is designed for fast-\/paced multiplayer gameplay, using UDP for minimal latency. ~\newline
 Each message is self-\/contained and includes its own header specifying type and size.

\hypertarget{md_docs_2server__protocol_autotoc_md20}{}\doxysection{\texorpdfstring{3. Protocol Overview}{3. Protocol Overview}}\label{md_docs_2server__protocol_autotoc_md20}
Clients communicate with the game server using UDP datagrams. ~\newline
 Each datagram contains exactly one packet consisting of a {\bfseries{\doxylink{struct_packet_header}{Packet\+Header}}} followed by a {\bfseries{payload}} specific to the message type.

Since UDP is unreliable, clients and servers {\bfseries{MUST}} implement their own sequencing and resynchronization mechanisms using the {\ttfamily sequence\+\_\+number} field provided in several packet types.

Typical communication flow\+:


\begin{DoxyEnumerate}
\item The client sends a {\ttfamily Player\+Info} packet to identify itself.
\item The server responds with a {\ttfamily New\+Player} packet.
\item Both sides exchange movement, projectile, and event packets continuously during gameplay.
\item A {\ttfamily Heartbeat} packet is exchanged periodically to maintain connection state.
\end{DoxyEnumerate}

\hypertarget{md_docs_2server__protocol_autotoc_md51}{}\doxysection{\texorpdfstring{7. Message Semantics}{7. Message Semantics}}\label{md_docs_2server__protocol_autotoc_md51}

\begin{DoxyItemize}
\item The server {\bfseries{MUST}} validate the {\ttfamily size} field of all packets. ~\newline

\item Clients {\bfseries{MUST NOT}} send unknown or malformed packet types. ~\newline

\item Sequence numbers {\bfseries{SHOULD}} be used to detect dropped or out-\/of-\/order UDP packets. ~\newline

\item The server {\bfseries{MAY}} ignore packets received out of sequence. ~\newline

\item The client {\bfseries{MUST}} periodically send a {\ttfamily Heartbeat} packet to prevent timeout. ~\newline

\end{DoxyItemize}
