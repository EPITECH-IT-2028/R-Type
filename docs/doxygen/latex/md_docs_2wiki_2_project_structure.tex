\begin{DoxyItemize}
\item {\bfseries{{\ttfamily .github/}}}\+: Contains Git\+Hub Actions workflows for continuous integration and deployment.
\item {\bfseries{{\ttfamily client/}}}\+: Houses all the code and resources specific to the R-\/\+Type client application. This includes rendering logic, asset management, client-\/side networking, and the main client application entry point.
\item {\bfseries{{\ttfamily core/}}}\+: This directory contains shared code that is utilized by both the client and the server. It includes common utilities, error handling, and the foundational networking components.
\item {\bfseries{{\ttfamily docs/}}}\+: Stores project documentation, including detailed server setup guides, network protocol specifications, and the Git\+Hub Wiki markdown files.
\item {\bfseries{{\ttfamily game\+\_\+engine/}}}\+: This is where the core Entity-\/\+Component-\/\+System (ECS) framework is implemented. It defines the base classes and managers for entities, components, and systems, along with specific implementations for various game mechanics.
\item {\bfseries{{\ttfamily server/}}}\+: Contains all the code and logic for the R-\/\+Type game server. This includes game state management, server-\/side networking, and specific game entities like enemies, players, and projectiles.
\item {\bfseries{{\ttfamily .venv/} \& {\ttfamily .venv-\/docker/}}}\+: These directories typically contain Python virtual environments, likely used for managing development tools, Conan, or Docker-\/related scripting.
\item {\bfseries{Root Files}}\+:
\begin{DoxyItemize}
\item {\ttfamily build.\+sh}\+: A shell script to automate the build process.
\item {\ttfamily CMake\+Lists.\+txt}\+: The primary CMake configuration file for the entire project.
\item {\ttfamily conanfile.\+txt} \& {\ttfamily conanfile\+\_\+server.\+txt}\+: Conan recipe files specifying project dependencies.
