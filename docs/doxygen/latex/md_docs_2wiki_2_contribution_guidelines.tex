We welcome and appreciate contributions to the R-\/\+Type project! By contributing, you help make this project better for everyone. Please take a moment to review this document to understand how to contribute effectively.\hypertarget{md_docs_2wiki_2_contribution_guidelines_autotoc_md71}{}\doxysection{\texorpdfstring{How to Contribute}{How to Contribute}}\label{md_docs_2wiki_2_contribution_guidelines_autotoc_md71}

\begin{DoxyEnumerate}
\item {\bfseries{Fork the Repository}}\+: Start by forking the main R-\/\+Type repository to your Git\+Hub account.
\item {\bfseries{Clone Your Fork}}\+: Clone your forked repository to your local machine. {\ttfamily bash git clone \href{https://github.com/YourUsername/CPP-Advanced_C---R-Type.git}{\texttt{https\+://github.\+com/\+Your\+Username/\+CPP-\/\+Advanced\+\_\+\+C-\/-\/-\/\+R-\/\+Type.\+git}} cd CPP-\/\+Advanced\+\_\+\+C-\/-\/-\/\+R-\/\+Type }
\item {\bfseries{Create a New Branch}}\+: Create a new branch for your feature or bug fix. Use a descriptive name (e.\+g., {\ttfamily feature/add-\/new-\/enemy-\/type}, {\ttfamily bugfix/fix-\/player-\/collision}). {\ttfamily bash git checkout -\/b your-\/branch-\/name }
\item {\bfseries{Make Your Changes}}\+: Implement your feature or fix the bug. Ensure your code adheres to the project\textquotesingle{}s coding style and conventions.
\item {\bfseries{Test Your Changes}}\+: If applicable, add or update unit tests to cover your changes. Ensure all existing tests pass.
\item {\bfseries{Commit Your Changes}}\+: Write clear, concise commit messages. A good commit message explains {\itshape what} was changed and {\itshape why}. {\ttfamily bash git commit -\/m "{}feat\+: Add new enemy type with unique attack pattern"{} }
\item {\bfseries{Push to Your Fork}}\+: Push your local branch to your forked repository on Git\+Hub. {\ttfamily bash git push origin your-\/branch-\/name }
\item {\bfseries{Create a Pull Request (PR)}}\+: Go to the original R-\/\+Type repository on Git\+Hub and create a new Pull Request from your forked branch. Provide a detailed description of your changes, including any relevant issue numbers.
\end{DoxyEnumerate}\hypertarget{md_docs_2wiki_2_contribution_guidelines_autotoc_md72}{}\doxysection{\texorpdfstring{Coding Style and Conventions}{Coding Style and Conventions}}\label{md_docs_2wiki_2_contribution_guidelines_autotoc_md72}

\begin{DoxyItemize}
\item {\bfseries{Follow Existing Style}}\+: Please adhere to the existing coding style and conventions found in the codebase. This includes naming conventions, indentation, brace style, and comment formatting.
\item {\bfseries{C++17 Standard}}\+: All new code should be compatible with the C++17 standard.
\item {\bfseries{Clear and Readable Code}}\+: Write code that is easy to understand and maintain.
\item {\bfseries{Comments}}\+: Add comments where necessary to explain complex logic or non-\/obvious decisions. Avoid commenting on obvious code.
\end{DoxyItemize}\hypertarget{md_docs_2wiki_2_contribution_guidelines_autotoc_md73}{}\doxysection{\texorpdfstring{Testing}{Testing}}\label{md_docs_2wiki_2_contribution_guidelines_autotoc_md73}

\begin{DoxyItemize}
\item {\bfseries{Unit Tests}}\+: For new features or bug fixes, please include relevant unit tests. This helps ensure the stability and correctness of the codebase.
\item {\bfseries{Build System}}\+: Ensure your changes build successfully using CMake and Conan.
\end{DoxyItemize}\hypertarget{md_docs_2wiki_2_contribution_guidelines_autotoc_md74}{}\doxysection{\texorpdfstring{Pull Request Guidelines}{Pull Request Guidelines}}\label{md_docs_2wiki_2_contribution_guidelines_autotoc_md74}

\begin{DoxyItemize}
\item {\bfseries{Descriptive Title}}\+: Your PR title should briefly summarize the changes.
\item {\bfseries{Detailed Description}}\+: Provide a clear and concise description of your changes. Explain the problem your PR solves, how it solves it, and any potential impacts.
\item {\bfseries{Refer to Issues}}\+: If your PR addresses an existing issue, please link to it (e.\+g., {\ttfamily Fixes \#123}, {\ttfamily Closes \#456}).
\item {\bfseries{Code Review}}\+: Be prepared for constructive feedback during the code review process. Address comments and make necessary adjustments.
\end{DoxyItemize}\hypertarget{md_docs_2wiki_2_contribution_guidelines_autotoc_md75}{}\doxysection{\texorpdfstring{Reporting Bugs}{Reporting Bugs}}\label{md_docs_2wiki_2_contribution_guidelines_autotoc_md75}
If you find a bug, please open an issue on the Git\+Hub issue tracker. Provide as much detail as possible, including\+:


\begin{DoxyItemize}
\item A clear and concise description of the bug.
\item Steps to reproduce the behavior.
\item Expected behavior.
\item Screenshots or error messages, if applicable.
\item Your operating system and compiler version.
\end{DoxyItemize}

Thank you for your contributions!
